# Getting Started

This page shows the common construction patterns for unified grids. Examples are
written as ordinary Julia snippets; they are not doctests because many projects
customize backends, halo sizes, and coordinate conventions.

## Analytic Mapped Grids

Use `MappedGrid` when coordinates are generated by smooth mapping functions.
The mapping functions receive time, computational indices, and a parameter
object.

```julia
using CurvilinearGrids
using KernelAbstractions: CPU

nx, ny = 32, 24
nhalo = 2
params = (; nx, ny, ax=0.03, ay=0.02)

xmap(t, xi, eta, p) =
  (xi - 1) / p.nx + p.ax * sinpi(2 * (eta - 1) / p.ny)
ymap(t, xi, eta, p) =
  (eta - 1) / p.ny + p.ay * sinpi(2 * (xi - 1) / p.nx)

grid = MappedGrid(
  xmap,
  ymap,
  params,
  (nx, ny),
  nhalo;
  backend=CPU(),
  coordinate_system=CartesianCS(),
  basis=CartesianBasis(),
  conserved_metric_scheme=CurvatureCorrectedReconstruction(),
  cache_mode=:eager,
)
```

For 3D analytic maps, `ADThomasLombardMetric()` can be selected when the goal is
to combine AD-evaluated mapping derivatives with Thomas-Lombard conservative
metric algebra. This is currently the recommended high-fidelity option for 3D
analytic mapped grids on CPU.

```julia
grid3 = MappedGrid(
  xmap3,
  ymap3,
  zmap3,
  params3,
  (nx, ny, nz),
  nhalo;
  diff_backend=AutoForwardDiff(),
  conserved_metric_scheme=ADThomasLombardMetric(),
  cache_mode=:eager,
)
```

## Discrete Grids From Imported Coordinates

Use `DiscreteGrid` when coordinates come from arrays. This is the current viable
path for CAD-generated or imported CFD meshes because the input is a set of
nodes, not a differentiable mapping.

```julia
grid = DiscreteGrid(
  xnodes,
  ynodes,
  nhalo;
  halo_coords_included=true,
  coordinate_system=CartesianCS(),
  basis=CartesianBasis(),
  interpolation=:linear,
  conserved_metric_scheme=CurvatureCorrectedReconstruction(),
  cache_mode=:eager,
)
```

The current interpolation-backed `DiscreteGrid` path supports linear
interpolation. AD on a discrete grid would differentiate the interpolation
surrogate, not the original CAD geometry. Conservative discrete metric
reconstruction therefore remains important for sampled grids.

## Coordinate-System Traits

Coordinate systems are selected with traits:

```julia
CartesianCS()
CylindricalCS()
SphericalCS()
AxisymmetricCS{:x}()
AxisymmetricCS{:y}()
CurvilinearCS()
```

Mapped and discrete grids also carry a basis trait:

```julia
CartesianBasis()
SphericalBasis()
```

The coordinate-system trait controls conservation measure factors. For example,
axisymmetric and cylindrical grids include `2*pi*r` factors in conservative
volumes and face metric vectors, while spherical grids use spherical basis
scales.

## Basic Geometry Access

```julia
I = first(grid.iterators.cell.domain)

q_cell = centroid(grid, I)
q_face = face_coordinate(grid, I.I, :ihi)

V = cellvolume(grid, I)
geom = face_flux_geometry(grid, I.I, :ihi)

S = geom.metric_vector
n = geom.normal
A = geom.area
```

For mapped and discrete grids, `face_flux_geometry` is the preferred public API
for solver-facing face geometry.
